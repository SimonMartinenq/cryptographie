Determine if a number is prime

What is a prime number ?

A natural number (greater than or equal to 2) is a prime number if it has exactly 2 divisors: 1 and itself. Example : 2, 3, 5, 7, 11, 13, 17, 19 ... are prime numbers. There is an infinity of them.

Naive method

To determine whether a natural number n greater than or equal to 2 is a prime, we must search for a divisor of n among successive primes (2, 3, 5, 7, 11 ...) up to the value √n. Indeed, if n admits no divisor among the successive primes up to the value √n, it will also admit none between √n and n because the divisors of a number go in pairs: one between 2 and √n, and the other between √n and n. If n has no divisors among successive primes up to the value √n, then it is a prime.

Sieve of Eratosthenes

In the sieve of Eratosthenes, which contains the numbers from 1 to 100, we crossed out successively the multiples of 2, those of 3, those of 5 and those of 7 (112 > 100), to obtain the list of the prime numbers lower than 100. 1 is considered as not being a prime number.

tips: know the first prime numbers

Compute PGCD

Definition

The Greatest Common Divisor (GCD) between two or more numbers is the largest natural number that simultaneously divides all of these numbers.

Euclide algorithm

The following table shows the calculation of the PGCD of 21 and 15. The Euclidean division of a = 21 and b = 15 is performed: the quotient is 1 and the remainder 6. The algorithm continues with a = b and b = the previous remainder, until finding a zero remainder. The algorithm then stops and returns the last non-zero remainder found, here r 3 = 3 which is indeed the PGCD of 21 and 15 :

21 = 15 x 1 + 6
15 = 6 x 2 + 3
6 = 3 x 2 + 0

Determine the prime factorisation of a number, deduce the number of divisors

Method

To date, there is no efficient method to decompose a number into prime factors. It is on this problem that the RSA cryptographic system is based. It is thus necessary "stupidly" to divide the chosen number by the factors which divide it, until obtaining only prime factors.

Find the inverse of one number:

Methode

Condition

First of all, before calculating the inverse of a number, you have to check if it is invertible. For this, we ask ourselves if the PGCD of a and n is equal to 1

Calculation

To find the inverse of a number in Z/nZ we use the Extended Euclidean algorithm (see above). The component "a" being the "n" and "b" being the number whose inverse is sought in Z/nZ. The inverse is the Vk component of the algorithm

Find liste of invetible in Z/nZ

Definition

In other words find all element a such that GCD(a,n)=1 otherwise written a ∧ n = 1

Methode

From what we have just seen, it seems long if we have to calculate the inverse of each number. But don't worry there are some techniques/properties to go faster.

Property

- 0 is never invertible

- 1 is always invertible and his inverse is 1

- Note the symmetry of the list

Execution

Let's see the execution through an example. Let's take Inv(Z/14Z) and make an array from 1 to 13. We know that 1 is invertible and that its inverse is 1. Then before calculating the inverse, we check that the numbers are invertible. For example for 2 : PGCD(2,14) != 1, so 2 is not invertible. We continue like that until the half of n. Because 3 = -11 so Inv(3) = Inv(-11) <==> Inv(11)=-Inv(3) (Symmetry of inverses). This symmetry is valid for non-invertible numbers: in our case 2 is not invertible so 12 is not either. One last thing to note: If the inverse of 3 is 5 for example then the inverse of 5 is 3.

Compute number of type: x^p[n]

Method

We need to decompose the power "p" of our number "x" into powers of 2 (column 1). For exemple: 23 = 2^0 + 2^1 + 2^2 + 2^4 = 1+2+4+16

Then we calculate our number x to the corresponding power of 2. For the first line it's 1, but for the following lines it's the same as doing our previous remainder to the square.

For the third column it is sufficient to perform the modulo "n" of the second column at each step. This reduces the calculation for column 2.

The last column indicates whether the power of 2 makes up our power "p"

Solve equation of the type: a * x = b in Z/nZ

Method

Case 1: a is invertible

In this case it's simple, it comes back to an "ordinary" equation: x = b * Inv(a)

Case 2: a is not invertible but b is a multiple of GCD(a,n)

In this case, the formula for x is: b/a + k * n/a, k ∈ {0,...,a-1}. If the value obtained with a case is not integer, it is not a solution.

Case 3: a is not invertible and b is not a multiple of GCD(a,n)

Sorry, there is no solution
